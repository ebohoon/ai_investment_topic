Vercel 배포 체크리스트 (이 저장소는 모노레포)

[필수] Root Directory
  Vercel → Project → Settings → General → Root Directory
  → 반드시 "비움" (저장소 루트 = ai_investment_topic 클론 최상위)
  → client 로 두면 안 됩니다.

  증상: Install 단계에서 패키지가 ~70개만 설치됨 (client만 설치된 것)
  정상: workspaces 때문에 패키지가 훨씬 많이 설치됨 (수백 개 근처)

[필수] Build / Output 오버라이드 끄기
  Settings → Build & Development Settings
  → Build Command, Output Directory, Install Command 의 Override 를 끄거나
  → 저장소 루트의 vercel.json 과 동일하게 맞추기

[환경 변수]
  OPENAI_API_KEY, MONGODB_URI (선택) 등은 Vercel Environment Variables 에 등록

이유: api/recommend.ts 와 server 코드는 저장소 루트에 있습니다.
      Root 를 client 로 하면 해당 파일이 배포에 포함되지 않고, 빌드 경로도 깨집니다.
