Vercel 배포

[권장] Root Directory = 비움 (Git 저장소 루트)
  - 루트 package.json + package-lock.json 이 함께 배포됨
  - vercel.json: build npm run build -w aicc-client, output client/dist
  - API: client/api/recommend.ts

[가능] Root Directory = client
  - client 폴더만 올라가도, 그 안에 server 소스 + api 가 포함됨
  - client/package.json 에 API용 의존성(openai, zod, mongodb, @vercel/node) 포함
  - client/vercel.json: build npm run build, output dist
  - 이 경우 루트 vercel.json 은 사용되지 않음

이전 에러(client/client/package.json)는 Root 가 client 인데
빌드 명령이 또 ./client 를 붙여서 생긴 것이었습니다.
지금은 Root 가 client 일 때는 client/vercel.json 이 npm run build 만 사용합니다.

로컬 API .env 위치: client/server/.env (이전 server/.env 는 여기로 옮기세요)

최신 코드(입력 검증 등)가 사이트에 안 보일 때:
  - Deployments → 최신 커밋으로 Redeploy
  - 브라우저 강력 새로고침 (Ctrl+Shift+R)
  - index.html 은 캐시하지 않도록 vercel.json headers 설정됨

환경 변수 넣었는데도 OPENAI_API_KEY 오류가 나면:
  - 이름이 정확히 OPENAI_API_KEY 인지 (대소문자, 값 앞뒤 공백·따옴표 없음)
  - 접속 URL이 Production 인데 변수는 Preview 에만 있거나 그 반대인지 → 둘 다 체크하거나 맞는 쪽에만 넣기
  - 저장 후 Deployments → 최신 배포 ⋯ → Redeploy (변수 추가 후 재배포 필수)
  - 런타임은 OPENAI_API_KEY 등을 명시적 process.env 로도 읽음 (Vercel 번들에서 동적 env[key] 가 비는 경우 대비)
